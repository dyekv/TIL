## AWS Shared Responsibility Model (責任共有モデル)

アプリケーションの保護について、ユーザとAWSの両者で責任を負う

アプリケーションを分割し、AWSが責任を負う部分とユーザが責任を負う部分に分ける

### AWSが管理する部分

- Physical（物理デバイス）
  - データセンターへは誰の訪問も許可しない
- ネットワーク
  - 専用のネットワークプロトコルで保護
- ハイパーバイザー
  - Xen
  
### ユーザが管理する部分

- Guest OS
  - EC2のOSなど
- アプリケーション
- ユーザデータ

ここについてはAWS側からアクセス、メンテナンスすることはできない
